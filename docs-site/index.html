Pulse Pager is open source uptime monitoring with an API-first design. It watches your endpoints, opens incidents when they break, notifies your team, and publishes status pages.

What Pulse Pager does

Monitors

HTTP checks with method, headers, body, expected status, latency and body assertions, failure thresholds, and multi-region dispatch.

Incidents

Failures open incidents automatically and close on recovery. Annotate the timeline or close manually as an operator override.

Notifications

Slack, Discord, email, generic webhook, and more. Test a channel before you rely on it.

Status pages

Publish a branded public status page with uptime history and recent incidents, served from a public-safe projection.

Teams and roles

Per-org membership with owner, admin, member, and viewer roles. Invite by email; roles map straight to API permissions.

API keys

Per-org, role-scoped keys (pulse_sk_...). Create one, copy a curl, and run it against your own org in minutes.

Getting started

The fastest way to your first check:

  1. Sign in with Google or GitHub. There is no password and no signup form.
  2. Your org is created for you on the Free plan, no card needed, with you as the owner.
  3. Create your first monitor in the app: give it a URL, pick how often to check, and save. The first result shows up in under two minutes.

Prefer to work headless? Everything the app does is also available over the REST API, so you can run Pulse Pager entirely from scripts or CI, no UI needed.

See Authentication for keys and roles, and the API reference for every operation.
